
# Flake-based NixOS Config

## Structure

```text
.
├── flake.nix                  # Main flake config
├── flake.lock                 # Pin versions of dependencies
├── hosts/
│   └── x1/
│       ├── configuration.nix          # x1's system config
│       └── hardware-configuration.nix # generated by nixos-generate-config
```

## Rebuilding System

To apply system changes for the x1 host:

```bash
sudo nixos-rebuild switch --flake .#x1
```

If you're setting this up on a new machine:

- Clone this repo.
- Make a new entry in flake.nix under nixosConfigurations.
- Add a `configuration.nix` and `hardware-configuration.nix` in hosts/<hostname>/.
    - You will want to copy from the new machine. This is the only heavy lifting.

Run the rebuild command above with the new host name.

## Key Concepts

- flake.nix: Central entry point. Defines all host machines.
- username is passed explicitly as a module argument (see userModule in flake.nix).

# Multiple Monitor Setup

To manage multiple monitor layouts smoothly:

    - Run `arandr` to visually configure your desired monitor setup.
    - Once your layout looks good, save the configuration:
        - Click *Layout > Save As*
        - Save two profiles:
            - `docked` (for when your external monitor(s) are connected)
            - `undocked` (for when you’re using just the laptop screen)
        - Alternatively, save layouts via autorandr:
        ```bash
        autorandr --save docked
        autorandr --save undocked
        ```
    - These profiles are used by autorandr to automatically switch layouts:
        - `autorandr --change docked` applies the docked layout.
        - `autorandr --change undocked` applies the undocked layout.
    - You can manually switch layouts with the following keybindings:
        - `Mod + F11` — switch to docked layout
        - `Mod + F12` — switch to undocked layout
